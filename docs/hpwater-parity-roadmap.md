# HPWater Parity Roadmap

Source target: https://github.com/AshenOneArt/HPWater, inspected at upstream commit `1253e5b`.

This document is the implementation contract for porting HPWater's Unity HDRP water pipeline into VibeEngine. The current VibeEngine implementation now has the first dedicated HPWater deferred path: a dynamic HPWater component, CPU wave mesh updates, a multi-scale directional spectral ocean layer with matching GBuffer normals and strict spectrum readback diagnostics, a GPU R16F fluid height ping-pong texture driven by an OpenGL compute wave update, HPWater-style top-down water/scene height-field resources for fluid obstacle boundaries, a dedicated water GBuffer, an explicit water mask, a scene-depth pyramid for refraction, water depth and normal merge passes that write valid HPWater surface data back into the main scene depth and normal buffers, a full-resolution refraction payload with HPWater-style world-space refracted rays projected into NDC for depth-pyramid marching and strict hidden readback diagnostics, a half-resolution volumetric accumulation/filter/composite path with an explicit low-resolution scene/camera motion-vector texture for temporal rejection plus serialized HPWater-style temporal/filter/shadow parameters and point/spot punctual volume scattering, a full-resolution caustic energy texture that can be filtered and consumed by both water composite and volume lighting, a directional-light cascade water atlas capture pass with HPWater-style caustic GBuffer absorption/scatter RT, an OpenGL compute caustic irradiance path with four-channel R32UI atomic accumulation plus RGBA16F RGB resolve that samples the real light-space cascade atlas, an HPWater-style compute-first caustic denoise filter with local shared-memory halo staging, a water-local SSR hierarchy slot with exported hit/confidence diagnostics, HDRP-style SSR/probe/sky hierarchy weight consumption, automatic launcher reflection-probe coverage that verifies non-zero probe and sky fallback hierarchy weights, automatic launcher two-probe coverage that verifies non-zero secondary-probe binding and blend under the same hierarchy diagnostics, shader-produced point/spot punctual light contribution diagnostics, and serialized BSDF controls for environment reflection, macro scatter, thin SSS, backlit transmission, forward scatter, preintegrated FGD LUT sampling, GGX energy compensation, HPWater-style exit Fresnel attenuation of the water body term, Unity HDRP `GetSpecularDominantDir` specular environment shaping, and HPWater-style upward multi-bounce reflection shaping. It is not feature-complete.

## Current VibeEngine Coverage

- `HPWaterComponent` creates a runtime water mesh and simulates a visible wave field on CPU.
- `HPWaterComponent` now drives ocean displacement and matching GBuffer normals, giving the ocean a visible multi-scale wave response even when mesh tessellation is coarse. The inspected HPWater source preview at `1253e5b` exposes the real-time wave-equation and shader-graph style water stack but not a directly portable FFT/Tessendorf implementation, so VibeEngine now owns its Tessendorf-style GPU spectral ocean foundation. The spectrum pipeline generates a bit-reversed animated Phillips/JONSWAP frequency spectrum, runs horizontal and vertical radix-2 inverse FFT butterfly passes through RGBA16F ping-pong textures, then resolves the spatial height field into the existing RGBA16F normal/height payload consumed by `HPWaterGBuffer.shader`; the previous direct k-grid summation remains as a shader fallback only. CPU mesh displacement, the GPU spectrum texture, and the GBuffer fallback share serialized wind-spectrum shaping controls: wind speed, directional spread, swell persistence, and short-wave fade.
- `Water.shader` provides a visible HPWater-inspired surface material.
- `HPWaterGBuffer.shader` renders water into a dedicated three-target resource set: normal/roughness, scatter/thickness, and absorption/foam.
- `HPWaterDepthMerge.shader` writes valid HPWater surface depth into the main scene depth buffer with hardware `LEQUAL` depth testing, while keeping the refraction depth pyramid opaque-only by building it before the merge.
- `HPWaterNormalMerge.shader` writes HPWater normal/roughness into the main scene GBuffer RT1 at water-covered pixels, matching HPWater's prepass-normal behavior for later screen-space passes.
- `HPWaterMask.shader` builds a full-resolution R8 water coverage mask from the dedicated HPWater depth buffer as the current OpenGL equivalent of HPWater's stencil isolation.
- `HPWaterDepthPyramid.shader` builds an opaque scene-depth mip chain for HPWater refraction.
- `HPWaterComposite.shader` performs Hi-Z-assisted full-resolution refraction payload generation and final water composite, including HPWater-style world-space refracted ray construction, NDC projection, exponential depth-pyramid marching, frame-indexed IGN step jitter, and tunable max refraction cross distance / thickness offset controls.
- `HPWaterVolume.shader` performs a first half-resolution volume accumulation pass for in-scattering, transmittance, and refracted depth. It now maps each half-resolution fragment back to the matching full-resolution HPWater GBuffer texel center, matching HPWater's low-resolution volume input convention, and treats `RefractMeta.w` as thickness rather than a hard water-valid flag so depth-merged water pixels still accumulate volume. Its volume loop now consumes HPWater's `_MaxCrossDistance`-style cap through the serialized `MaxRefractionCrossDistance`, building a no-linear volume ray length and a Beer-Lambert-derived dynamic shadow distance before sampling the active directional CSM through HPWater-style serialized shadow parameters: shadow softness, minimum filter size, blocker sample count, and filter sample count, with frame-indexed cascade dither at split boundaries. It also consumes the same scene point/spot light arrays as the surface composite path so local lights contribute to volumetric water scattering, not only final-surface specular/body lighting. Directional, point, spot, and rectangle-area volume scattering now use HPWater's serialized `_PhaseG` and albedo-driven phase blend shape, interpolating the scatter phase toward isotropic scattering as the scattering albedo rises.
- `HPWaterVolumeMotionVector.shader` generates an explicit half-resolution RG16F volume velocity texture from current/previous view-projection reprojection of the main scene GBuffer world-position payload, falling back to the refracted water world payload where a scene position is unavailable. The scene-position path now also accepts a Scene-tracked rigid-object world offset derived from active mesh renderer UUIDs and previous-frame transforms, so moving rigid objects can contribute a first object-motion term instead of being treated as static camera-only history. The stored vector follows HPWater's sampled convention (`historyUV = currentUV - motionVector`). Diagnostics distinguish the explicit velocity texture, scene-motion-vector source, object-motion tracked/matched/moving counts, and averaged world offset.
- The object-motion bridge now also uploads a small per-object motion field: up to eight moving rigid mesh candidates, each represented by a current world-space influence sphere and its own previous-frame offset. `HPWaterVolumeMotionVector.shader` deterministically selects the nearest matching object slot for each low-resolution volume pixel before falling back to the averaged offset, moving the temporal volume path closer to HDRP's per-pixel object motion-vector buffer while keeping the current GBuffer layout unchanged. The previous-frame object snapshot is captured once per logical `Scene::OnUpdate()` instead of at the end of every render pass, so editor Scene View and Game View cannot erase motion by rendering the same scene twice in one frame.
- `HPWaterVolumeTemporal.shader` performs low-resolution temporal reprojection/history blend before spatial filtering, using the explicit motion-vector texture to address history UVs and apply HPWater's velocity-weighted history rejection while retaining current/previous view-projection reprojection as fallback, plus depth rejection and HPWater/HDRP-style 3x3 current-neighborhood color/transmittance clamp. `HPWaterComponent` now serializes the HPWater defaults for temporal blend, motion-vector usage, velocity scale, temporal depth rejection, and depth threshold.
- `HPWaterVolumeFilter.shader` performs the first multi-iteration depth-aware a-trous-style spatial filter over the half-resolution volume buffers. `HPWaterComponent` now serializes the spatial filter enable flag, iteration count, depth-aware flag, and depth sensitivity so the pass no longer runs a fixed three-iteration filter.
- `HPWaterVolumeUpsample.shader` resolves filtered half-resolution volume buffers into full-resolution joint-bilateral volume textures and uses the same water-mask/depth validity rule as accumulation rather than rejecting zero-thickness metadata.
- `HPWaterCaustic.shader` performs a first full-resolution caustic energy accumulation from HPWater normals, thickness, absorption, mask, scene depth, and directional light parameters, including a serialized single-channel/RGB dispersion mode inspired by HPWater's `_Is_Use_RGB_Caustic` and `_CausticDispersionStrength` controls. When compute caustics are available in the square cascade atlas it projects receivers into cascade space, clamps atlas tile samples to half-texel bounds, fades tile edges, and blends into the next cascade near split boundaries.
- `HPWaterCausticFilter.comp` is the primary HPWater-style caustic denoise/filter path. It runs in 16x16 compute groups, stages a 18x18 shared-memory color tile with one-pixel halo/corners, and applies the upstream 3x3 a-trous luminance edge rule (`ComputeEdgeWeight`, `1/16, 2/16, 4/16`, `atrousLuminanceWeight = 0.5`) with HPWater depth and explicit water-mask guards. `HPWaterCausticFilter.shader` remains the fullscreen fallback with the same kernel semantics.
- `HPWaterCausticAtlas.shader` captures water-only surface payloads from the four directional-light cascade views into a 2x2 atlas, reusing the current CSM light view-projection matrices as the first HPWater-style light-space caustic input resource. RT0 stores encoded normal and thickness/roughness, while RT1 stores sRGB-encoded absorption/scatter luminance so the fullscreen caustic pass can decode a HPWater-style light-space caustic GBuffer with 2x2 depth-aware weights.
- `HPWaterCausticIrradiance.comp` is the first OpenGL compute slice of HPWater caustics: it scatters refracted caustic energy from the HPWater GBuffer, explicit mask, scene depth, light parameters, the current light-space water atlas, and the active directional shadow-depth cascade array into four square R32UI atomic cascade-atlas irradiance textures. The compute pass reconstructs water world positions from the camera depth, selects the matching directional-light cascade from the current CSM split distances, projects the water point through the uploaded cascade VP matrix, samples the real 2x2 water atlas tile instead of synthesizing cascade tiles from screen UVs, performs a HPWater-style exponentially distributed refracted light ray march with frame-indexed R2 dither against the shadow-depth texture array, and now uses the reference-style water-cascade NDC start/end segment when atlas and shadow inputs are valid instead of relying only on the previous world-space depth-delta window. It gates caustic leakage with a serialized `_CausticShadowAlphaClipThreshold`-equivalent receiver visibility threshold, writes receiver irradiance in light-space cascade atlas UVs, and reserves the resolved compute alpha channel as NDC receiver-hit evidence while final caustic energy validity is taken from RGB. Hidden caustic diagnostics now require `HPWaterCausticComputeAnyNDCReceiverHit=1`, proving the enabled NDC path actually produced receiver hits rather than only having valid inputs. In RGB caustic mode it projects R/B receivers with HPWater-style per-channel eta offsets in the same atlas receiver space, applies atlas receiver edge falloff, and adds a conservative per-channel spectral energy weighting.
- `HPWaterCausticIrradianceResolve.comp` resolves the atomic R/G/B/A irradiance textures into a full-resolution RGBA16F texture that `HPWaterCaustic.shader` consumes during full-resolution caustic accumulation while still retaining the previous fullscreen path as the visible fallback.
- `HPWaterVolume.shader` can inject filtered caustic energy into the directional in-scattering term so caustics affect volumetric water lighting, not only the final surface composite.
- `HPWaterFluidHeightCapture.shader` captures the first top-down R16F water-height and scene-height inputs used by fluid dynamics. The virtual capture camera now follows HPWater's down-facing camera convention with world +Z as camera up, so captured texture V coordinates match `WorldPosToFluidUV()`. The water target now rasterizes the displaced HPWater surface height from an independent water-layer capture list rather than from the main-camera visible water list, while the scene target rasterizes actual opaque geometry world height with max blending. Water-height capture is constrained to the active water layer, and scene-height capture excludes that layer and transparent renderers, matching HPWater's `RenderFluidHeight(..., waterLayerMask/~waterLayerMask, RenderQueueRange.opaque)` behavior. Diagnostics now expose candidate-level capture filtering: mesh candidates, water-layer candidates, scene opaque candidates, water-layer skips, scene water-layer skips, transparent skips, and final water/scene draw counts. The CPU fallback uses the same layer-filtered water capture list when sampling the current HPWater dynamic mesh/spectrum height instead of only uploading a flat ocean height. The same 16-band directional spectrum is evaluated in `HPWaterGBuffer.shader` so fluid normals and spectral normals combine in the dedicated water GBuffer instead of only changing CPU vertices. `HPWaterComponent::FluidStartFrameBake` mirrors HPWater's `startFrameBakeProcess`: when enabled, matching resolution/box captures are reused after the first valid bake instead of redrawn every frame.
- `HPWaterFluidDynamics.comp` performs the primary GPU ping-pong wave equation update into an R16F height texture, with `HPWaterFluidDynamics.shader` kept as a fullscreen fallback. It now consumes HPWater-style normalized water-height and scene-height textures first, then falls back to the older R8 obstacle mask if the height fields are unavailable. Its wave equation, boundary damping, and source injection now match HPWater's neighbor-reflection update without VibeEngine-specific obstacle-contact damping, wave-speed clamp, or output-height clamp, while retaining HPWater's X/Y edge absorption formula and positive-only, 0.05-clamped Gaussian source impulse. The source path supports up to eight simultaneous sources and promotes opaque scene objects that actually write into the fluid obstacle height field into object-driven wave sources, so train/boat/gameplay meshes can generate fluid impulses before a dedicated gameplay event stream exists. Moving obstacle sources now reuse Scene's previous-frame object-center snapshot to emit an extra trailing wake impulse behind the current footprint, and they are kept as priority sources even when the eight-source budget is saturated, giving contact waves directional history instead of a stationary circular pulse. `HPWaterGBuffer.shader` samples the fluid height texture for fluid normal and foam contribution using HPWater-style world-to-fluid UVs with clamp sampling at the simulation box edge.
- `HPWaterComposite.shader` and `HPWaterVolume.shader` expose the first HPWater-style BSDF/scatter controls: Schlick Fresnel using water IOR 1.33, environment reflection intensity, HPWater's `_IndirectLightStrength` for diffuse indirect water body lighting, a HPWaterBSDFLibary-style 16-step exponential indirect scattering integration, macro volume scatter strength, thin-layer SSS, backlit transmission, forward scattering, preintegrated split-sum FGD LUT sampling, analytic FGD fallback, and GGX energy compensation. The direct diffuse body split now follows HPWaterBSDFLibary's component weighting shape: `diffR = T_entry * G_entry * S_volume`, `diffT = thinLayerSSS + backlitTransmission`, with HPWater-style `G_sss`, `G_backlit`, SSS/backlit path scales, scatter phase, and Beer-Lambert transmittance. The composite light loop now also splits local point/spot/rectangle area lights into HPWater-style macro, thin-SSS, and backlit body buckets instead of feeding them through one `NdotL`-weighted aggregate, and direct water specular uses HPWater's `saturate(NdotL * 5)` self-occlusion gate. The point/spot branch now writes its own full-resolution diagnostics MRT with punctual specular, macro body, SSS/backlit body, and path-hit channels so hidden light diagnostics can prove shader-produced punctual contribution separately from area lights. The composite pass now applies HPWater's full-resolution exit Fresnel step (`T_exit = 1 - F(NdotV)`) to the water body term before adding surface reflection/specular and foam. The forward-scatter blur now uses HPWater's `CalculateHPWaterMipLevel(depth, FORWARD_SCALING_FACTOR, scatterDensity, 6)` shape with serialized `ForwardScatterBlurDensity`, `MultiScatterScale`, and `_PhaseG = 0.80` controls. The composite pass now samples the active sky texture as an equirectangular reflection source, consumes a dedicated full-resolution SSR lighting buffer before environment fallback, consumes the reflection hierarchy in HPWater/HDRP order (SSR first, reflection probe second, sky fallback last), exports a full-resolution SSR diagnostics target for confidence/hit-mask/probe-weight/sky-weight validation, and the launcher runtime creates and bakes two scoped reflection probes in the camera fade zone so hidden diagnostics verify primary probe binding, secondary probe binding, non-zero probe blend, and sky fallback together. It can consume and distance-blend the two nearest baked reflection-probe cubemaps when available, applies HDRP-style box-projected cubemap directions from each probe's world center and box size, weights probes by influence volume before falling back to the sky/ground gradient, and shapes specular environment directions with Unity HDRP's `GetSpecularDominantDir` formula after converting VibeEngine's linear GGX roughness to perceptual roughness, then applies HPWater's upward multi-bounce reflection tweak before sky/probe sampling. Surface and volume water light loops now also accept serialized rectangular `AreaLightComponent` inputs, sorted by estimated HPWater bounds influence and evaluated against Unity HDRP's official 64x64 RGBA16F `_LtcData` GGX and Disney layers (`U = perceptual roughness`, `V = sqrt(1 - clamped NdotV)`, `RGBA = _m00_m02_m11_m20` inverse LTC matrix coefficients). The rectangle-area path now transforms unnormalized rectangle vertices with the sampled inverse LTC matrix, clips the quad against the local horizon using HDRP's 16-case `ClipQuadToHorizon` logic, and integrates the resulting 3-5 vertex polygon with vector form factors instead of using the previous `abs(edgeSum)` or sample-point approximation. It also mirrors HPWater's `_WaterDispersionStrength = 0.1` default with clamped refracted-UV long/mid/short color sampling, currently applied in the composite forward-scatter scene-color path as the closest VibeEngine equivalent to HPWater's deferred volume composite. These controls and bindings are serialized or exported in render diagnostics.
- `HPWaterSSR.shader` generates the first HPWater/HDRP-style `_SsrLightingTexture` equivalent before the water light loop consumes it: RGB stores the weighted screen-space reflection lighting and alpha stores the consumed reflection hierarchy weight, matching `EvaluateBSDF_ScreenSpaceReflection()` semantics where the light loop consumes RGB directly and uses alpha only for hierarchy fallback. The standalone SSR pass now performs coarse depth-pyramid marching, accepts explicit ray-vs-scene-depth sign crossings as well as near-surface samples, then performs six mip0 binary refinement steps at the hit interval before sampling scene color. The accepted hit now samples scene color through a roughness-driven cone footprint over the generated scene-color mip chain, so glossy water keeps sharp hits while rougher water gathers a wider reflected footprint before the result is preweighted into the SSR hierarchy slot. Its temporal resolve now has a dedicated full-resolution RG16F SSR motion-vector target generated from HPWater depth and current/previous view-projection, and history sampling prefers HPWater's `historyUV = currentUV - motionVector` convention before falling back to matrix reprojection. This makes the SSR pass robust enough to produce real screen-space hits in the hidden launcher diagnostic while preserving the HDRP-style hierarchy alpha contract. The final composite no longer carries a duplicate local SSR ray marcher; it consumes only the preweighted SSR lighting buffer when that buffer is valid, and diagnostics report that contract explicitly.
- `DeferredRenderer` exports HPWater GBuffer, explicit water mask, main scene depth-stencil water marking, scene-depth pyramid, refraction, volume, caustic, SSR lighting, SSR resolve diagnostics, SSR motion-vector history, and final composite diagnostics. Hidden texture probes now report average RGBA channels in addition to luminance/alpha, and `render_diagnostics.txt` mirrors the HPWater SSR hierarchy channels as named fields (`HPWaterSSRDiagnosticsAverageConfidence`, `HPWaterSSRDiagnosticsAverageHitMask`, `HPWaterSSRDiagnosticsAverageProbeHierarchyWeight`, and `HPWaterSSRDiagnosticsAverageSkyHierarchyWeight`), plus derived environment fallback, hierarchy-sum, unallocated-weight, any-hit, any-probe, any-sky, SSR-lighting luminance, SSR-lighting hierarchy-alpha, SSR-lighting contribution, `HPWaterSSRExplicitMotionVectorEnabled`, `HPWaterSSRMotionVectorHistoryEnabled`, SSR motion-vector texture id, readback flag, magnitude proxy, motion-vector BMP export fields, and SSR resolve diagnostics fields for roughness cone LOD, temporal history weight, disocclusion acceptance, and motion-vector mode. This makes SSR/probe/sky allocation, the dedicated SSR lighting handoff, and the current SSR roughness/temporal heuristic behavior machine-readable without BMP inspection or probe-block parsing. `Sandbox.exe --hpwater-motion-diagnostics` now loads `Assets/launcher.vscene`, enters play mode, creates a moving diagnostic rigid mesh, waits until the HPWater object-motion field is selected, writes `render_diagnostics.txt`, and exits without requiring a user screenshot. `Sandbox.exe --hpwater-fluid-filter-diagnostics` now creates water-layer, opaque-scene, transparent, and moving opaque diagnostic meshes, waits until water-layer/render-queue capture filters and the moving object wake-source path are externally provable, writes `render_diagnostics.txt`, and exits. `Sandbox.exe --hpwater-fluid-bake-diagnostics` reuses the same diagnostic geometry, forces `FluidStartFrameBake`, waits until the cached water/scene height captures are reused while remaining valid, writes `render_diagnostics.txt`, and exits without requiring fresh capture draw counts on the reuse frame. Strict hidden diagnostics no longer export only because the max-frame timeout elapsed; required HPWater conditions must be true. `Sandbox.exe --hpwater-volume-diagnostics` now loads launcher, waits for HPWater volume accumulation, temporal reprojection, spatial filtering, upsampling, full-resolution composite handoff, and non-empty raw/filtered/upsampled volume color/depth texture readbacks before exporting. `Sandbox.exe --hpwater-ssr-diagnostics` now loads launcher, forces SSR on, injects a temporary reflector, waits until the dedicated HPWater SSR lighting buffer, explicit SSR motion-vector texture readback/history, resolve diagnostics texture, and hierarchy diagnostics target are valid and consumed by composite, writes `render_diagnostics.txt`, and exits.
- The editor can create, edit, serialize, and diagnose HPWater entities, and auto-export readback BMPs without relying on user screenshots or the Render Debugger panel being open.

## HPWater Source Features Not Yet Ported

### Dedicated Water GBuffer

HPWater renders water into a dedicated 3-target GBuffer:

- GBuffer0: world normal plus roughness.
- GBuffer1: scatter color and thickness.
- GBuffer2: absorption color plus foam.
- Water depth is written into the main prepass depth so later screen-space passes can distinguish water surface depth from scene depth.
- Water normal/roughness is written into the main prepass normal buffer so SSR and other screen-space consumers see the water surface normal.
- Stencil marks water pixels for later full-screen passes.

VibeEngine now has the dedicated three-target HPWater GBuffer and diagnostics. It also merges valid HPWater surface depth back into the main scene depth buffer after building the opaque-only refraction depth pyramid, so later screen-space passes can distinguish water surface depth without causing HPWater refraction to self-intersect. During that merge, the main scene depth-stencil now receives the HPWater/HDRP-style water classification bit pattern `WaterSurface | TraceReflectionRay` (`0x28` / `40`) for water pixels that pass the scene-depth test. VibeEngine still generates a dedicated full-resolution R8 HPWater mask texture from the HPWater depth buffer as a texture-addressable pass-isolation path for fullscreen shaders that cannot directly sample platform stencil.

### Refraction Buffer

HPWater generates a full-resolution refraction texture where:

- XY stores refracted UV offset.
- Z stores refracted scene depth.
- W stores the water pixel mask.
- It can use either a cheaper normal-offset path or ray marching.
- The ray marching path uses the scene depth pyramid excluding water, exponential stepping, IGN jitter, and a max cross distance.

VibeEngine now writes a full-resolution refraction payload and metadata target during `HPWaterComposite.shader`. The current implementation builds a dedicated opaque scene-depth pyramid, computes a HPWater-style world-space refracted ray from the camera-to-water direction, water normal, flat-normal subtraction, and water-cross direction, projects the ray into NDC, then marches that 3D NDC line with exponential steps, depth-pyramid coarse sampling, and mip-0 binary refinement. The march has HPWater-style frame-indexed IGN step jitter plus serialized controls for sample count, max refraction cross distance, and thickness offset. Refraction, volume accumulation, and volume upsample now share the explicit HPWater mask. The opaque depth pyramid now remains valid across the deferred lighting pass until HPWater composite consumes it; HPWater composite uses merged scene depth only for foreground rejection and uses the opaque depth pyramid for water thickness, scene-world reconstruction, and refraction fallback. Hidden diagnostics export float-precision named refraction world/meta fields for world payload, ray length, UV, scene depth, normalized thickness, thickness presence, and derived screen-UV offset evidence. `Sandbox.exe --hpwater-refraction-diagnostics` now loads launcher and only exits once the NDC refraction path, non-empty world/meta textures, positive thickness, and non-zero refraction UV offset are all externally proven. Remaining refraction parity work is closer validation against HPWater's exact Unity projection conventions and edge-case tuning.

### Volumetric Water Lighting

HPWater uses low-resolution volumetric accumulation, then filters and composites it:

- Ray-marched water scattering and transmittance.
- Absorption and scatter coefficients from the HPWater GBuffer.
- Temporal reprojection with history color, transmittance, depth, motion vectors, depth rejection, and velocity weighting.
- A-trous spatial filtering.
- Depth-aware joint bilateral upsampling using HPWater's 2x2 gather-style low-resolution reconstruction weights.
- Final composite with full-resolution specular lighting and refraction.

VibeEngine now has a half-resolution `HPWaterVolume.shader` accumulation target with in-scattering, transmittance, and refracted depth, plus a temporal reprojection/history pass, a configurable depth-aware a-trous-style spatial filter, and a full-resolution joint-bilateral upsample pass. The accumulation pass now follows the HPWater reference more closely by mapping low-resolution volume pixels back to full-resolution HPWater GBuffer texel centers, then marching the refracted water ray with exponentially distributed samples and frame-indexed IGN jitter. It accumulates directional in-scattering, point/spot/area-light in-scattering, HPWater-style after-loop scene in-scattering, and transmittance through the water volume instead of relying on a single analytic approximation. The scene in-scattering term now mirrors `WaterVolumetrics()` by multiplying final refracted scene color by cumulative transmittance, scatter coefficient, no-linear ray length, directional light luminance, and HPWater's `lerp(shadowValue, 1, 0.3)` shadow relaxation. Because VibeEngine's depth-merge path can leave `RefractMeta.w` at zero even for valid water pixels, accumulation and upsample now use mask/depth for water validity and treat metadata alpha only as a thickness input with a fallback estimate. Hidden diagnostics now report non-empty half-resolution accumulation, history/filter, and full-resolution upsample textures instead of all-black volume targets. Its volume ray now mirrors HPWater's `_MaxCrossDistance` no-linear cap: `MaxRefractionCrossDistance` is passed into the volume shader, sampling uses the capped ray length, and directional shadow sampling uses a Beer-Lambert-derived dynamic shadow distance based on absorption plus scatter. Its direct scattering now mirrors HPWater's `WaterVolumetrics()` phase response by consuming the serialized `_PhaseG` value and blending the scatter phase toward isotropic scattering with `smoothstep(0, 0.5, scatteringAlbedo)`. The volume loop now has a HPWater-style `_HPWaterShadowParams` equivalent: serialized volume shadow softness, minimum filter size, blocker samples, and filter samples are routed through `DeferredRenderer` into a water-specific CSM visibility function. Both HPWater surface and volume shadow paths use frame-indexed interleaved-gradient cascade dither at split boundaries, matching HPWater's dithered split selection more closely than the generic renderer's linear cascade blend. The temporal/filter path now exposes HPWater-style serialized defaults for temporal blend (`0.9`), motion-vector usage, velocity rejection scale (`0.1`), temporal depth rejection and threshold (`0.5`), spatial filter enable, spatial filter iterations (`2`), spatial depth awareness, and spatial depth sensitivity (`100`). The temporal pass generates and consumes an explicit RG16F volume motion-vector texture when enabled, now sourcing camera/scene motion from the main GBuffer world-position RT before falling back to the water refraction payload; for active rigid mesh renderers it can additionally estimate previous world positions from the Scene's UUID-keyed previous-transform cache, upload up to eight current influence spheres, and select the nearest matching object slot per volume pixel before falling back to an averaged rigid-object offset. When the explicit vector is valid, history UVs are sampled with HPWater's `currentUV - motionVector` convention and velocity history weight uses HPWater's `saturate(1 - velocity * velocityScale)` form; when it is unavailable, the pass falls back to matrix reprojection. The pass rejects mismatched history with both center-depth and neighborhood-depth tests when enabled, clamps history color/transmittance to the current 3x3 neighborhood before blending, and the spatial filter can be skipped without breaking volume upsample because upsample falls back to temporal/raw volume data. The full-resolution upsample now follows HPWater's gather-style reconstruction more closely by resolving four low-resolution taps with bilinear weights multiplied by reciprocal depth-difference weights, exports explicit diagnostics for the gather/depth-aware path, and the final HPWater composite now directly samples those full-resolution upsampled volume textures instead of re-running the low-resolution 2x2 reconstruction inside the surface composite pass. `Sandbox.exe --hpwater-volume-diagnostics` is the dedicated launcher gate for this path: it requires the accumulation, temporal, motion-vector history, shadow-parameter, filter, upsample, and full-resolution composite flags plus non-empty raw/filtered/upsampled volume color/depth readbacks. True parity with HPWater/HDRP's real per-pixel object-ID velocity buffers, skinned motion-vector source, validation heuristics, shadowed punctual lights, and exact Unity projection conventions remains pending.

### Caustics

HPWater caustics are a separate compute-driven pipeline:

- Renders water cascade depth and water GBuffer atlases from directional-light cascade views.
- Reads the shadow cascade atlas / scene depth.
- Refracts light through water normals.
- Accumulates caustic irradiance with atomics.
- Supports single-channel and RGB dispersion modes.
- Applies denoise / filtering and passes caustic data into deferred water lighting.

VibeEngine now has the first caustic resource slices: `HPWaterCaustic.shader` writes a full-resolution RGBA16F caustic texture, supports a screen-space single-channel/RGB dispersion mode, consumes a compute-generated RGBA16F irradiance texture, `HPWaterCausticFilter.comp` performs the primary compute denoise/filter pass with HPWater-style 16x16 thread groups and 18x18 LDS halo staging, `HPWaterCausticFilter.shader` remains a fullscreen fallback with the same 3x3 a-trous luminance edge kernel, `HPWaterComposite.shader` consumes the filtered texture when available, `HPWaterVolume.shader` injects filtered caustic energy into volume scattering, and `HPWaterCausticAtlas.shader` captures water-only payloads from all four directional-light cascade views into a 2x2 atlas. The atlas now has a second HPWater-style caustic GBuffer RT for sRGB-encoded absorption/scatter, and the fullscreen caustic pass decodes it with 2x2 depth-aware atlas weights before applying receiver transmittance. `HPWaterCausticIrradiance.comp` now uses HPWater-style multi-writer atomic irradiance accumulation through four R32UI square cascade-atlas channel textures, samples the actual light-space water atlas by reconstructing world position and applying the current CSM cascade VP matrices, binds the active CSM shadow-depth texture array for a HPWater-style exponentially distributed refracted light ray march with frame-indexed R2 dither, gates leakage with a serialized `CausticShadowAlphaClipThreshold` defaulting to HPWater's 0.8, writes per-channel R/G/B caustic receivers in cascade atlas UV space instead of camera UV space, fades atlas receiver tile edges, and applies conservative per-channel spectral energy weights in RGB mode. `HPWaterCaustic.shader` projects the current receiver world position back into the matching cascade atlas, clamps samples to tile-safe half-texel bounds, fades tile edges, blends into the next cascade near split boundaries to reduce cascade seam discontinuities, and now uses HPWater GBuffer absorption/scatter plus receiver thickness as a transmittance and leak-reduction gate. ### Fluid Dynamics

HPWater includes a GPU wave equation system:

- Top-down virtual orthographic capture of water height.
- Top-down virtual orthographic capture of scene height.
- Ping-pong R16 height textures.
- Compute wave propagation with obstacle boundaries, edge damping, and impulse sources.
- Global wave height texture for water shaders.

The water-height target now comes from a water-layer-only displaced HPWater draw list that is independent of main-camera frustum culling, and its CPU fallback samples the same layer-filtered current HPWater dynamic mesh/spectrum height.
